# 📖 Αναλυτικός Οδηγός Prompts για Δημιουργία Πινάκων-Παιχνιδιών με AI & NotebookLM

Αυτός ο οδηγός περιέχει δύο εξαιρετικά αναλυτικά prompts που μπορείτε να αντιγράψετε και να χρησιμοποιήσετε απευθείας:
1. **Prompt 1 (Για Multimodal AI - Claude, ChatGPT, Gemini)**: Όταν δίνετε φωτογραφίες πινάκων (π.χ. από βιβλίο, σημειώσεις) ή ακατέργαστα κείμενα.
2. **Prompt 2 (Για NotebookLM)**: Όταν θέλετε να αντλήσει πληροφορίες αποκλειστικά από τα έγγραφα/πηγές (sources) που του έχετε φορτώσει, χωρίς ανάγνωση εικόνας.

---

## 📸 PROMPT 1: Για Πολυτροπικά AI (Claude 3.5 Sonnet, ChatGPT-4o, Gemini 1.5 Pro)
*Κατάλληλο για μετατροπή εικόνων (φωτογραφίες πινάκων, screenshots) ή κειμένων σε έτοιμο κώδικα.*

### Πώς να το χρησιμοποιήσετε:
1. Ανεβάστε τη **φωτογραφία του πίνακα** (ή επικολλήστε το κείμενο).
2. Αντιγράψτε και επικολλήστε το παρακάτω κείμενο στο πεδίο εντολής (prompt).

```text
Θέλω να λειτουργήσεις ως κορυφαίος Παιδίατρος, Πανεπιστημιακός Εκπαιδευτής και Senior Web Developer. 
Σου παρέχω μια εικόνα (ή κείμενο) ενός ιατρικού πίνακα ή θέματος. Θέλω να το αναλύσεις προσεκτικά και να το μετατρέψεις σε ένα δομημένο αντικείμενο JavaScript (JSON) για ένα διαδραστικό παιχνίδι συμπλήρωσης κενών σε ιατρικούς πίνακες.

Ακολούθησε ΑΥΣΤΗΡΑ τους παρακάτω κανόνες:

1. ΔΟΜΗ ΠΙΝΑΚΑ ΚΑΙ ΑΠΛΟΠΟΙΗΣΗ:
   - Μετάτρεψε τον πίνακα σε απλή, κατανοητή δομή γραμμών-στηλών.
   - Η πρώτη στήλη (αριστερά) πρέπει να είναι σταθερό κείμενο ("type": "text"), π.χ. τα κλινικά σημεία ή οι κατηγορίες.
   - Οι επόμενες στήλες πρέπει να είναι οι ζώνες τοποθέτησης ("type": "zone").
   - Κάθε ζώνη ("zone") πρέπει να έχει:
     * "id": ένα μοναδικό αναγνωριστικό (π.χ. "gcs_eye_4", "apgar_pulse_0").
     * "expected": το ακριβές κείμενο της σωστής απάντησης που πρέπει να τοποθετηθεί εκεί.

2. ΔΗΜΙΟΥΡΓΙΑ ΠΑΓΙΔΩΝ (DISTRACTORS) - ΚΡΙΣΙΜΟ:
   - Δημιούργησε μια λίστα "distractors" με ακριβώς 8 παγίδες (ούτε περισσότερες, ούτε λιγότερες).
   - Οι παγίδες πρέπει να είναι εξαιρετικά αληθοφανείς ιατρικοί όροι, συμπτώματα ή τιμές που σχετίζονται στενά με την Παιδιατρική και το συγκεκριμένο θέμα, ώστε να μπερδεύουν δημιουργικά τον παίκτη.
   - Πρέπει όμως να είναι κλινικά και επιστημονικά ΛΑΘΟΣ για τις συγκεκριμένες θέσεις του πίνακα. 
   - Αποφύγε εντελώς άσχετους όρους (π.χ. αν ο πίνακας αφορά Μηνιγγίτιδα, οι παγίδες πρέπει να σχετίζονται με νευρολογικά/λοιμώδη ευρήματα της παιδικής ηλικίας, όχι με ορθοπαιδική ή καρδιολογία).

3. ΠΑΘΟΦΥΣΙΟΛΟΓΙΑ & ΚΛΙΝΙΚΟΣ ΣΥΛΛΟΓΙΣΜΟΣ (pathophysiology):
   - Γράψε μια αναλυτική παράγραφο (100-150 λέξεις) στα Ελληνικά.
   - Πρέπει να εξηγεί τον παθοφυσιολογικό μηχανισμό της νόσου/κατάστασης, βοηθώντας τον φοιτητή να κατανοήσει λογικά γιατί εμφανίζονται αυτά τα ευρήματα, αντί να τα απομνημονεύσει.

4. ΚΛΙΝΙΚΗ ΑΝΑΛΥΣΗ (fullExplanation):
   - Γράψε μια εκτενή ιατρική ανάλυση (150-250 λέξεις) στα Ελληνικά.
   - Πρέπει να αναφέρει την κλινική σημασία του πίνακα, διαγνωστικά "red flags", όρια της μεθόδου και κατευθυντήριες οδηγίες αντιμετώπισης.

5. ΜΟΡΦΟΠΟΙΗΣΗ ΚΩΔΙΚΑ (JSON):
   - Επίστρεψε ΜΟΝΟ το έτοιμο αντικείμενο JSON, χωρίς εισαγωγικά κείμενα ή markdown σχόλια εκτός κώδικα, ώστε να μπορώ να το κάνω απευθείας αντιγραφή-επικόλληση.
   - Χρησιμοποίησε ακριβώς αυτή τη δομή:

{
  "id": "apgar_score", // Μοναδικό ID στα αγγλικά με πεζά
  "chapter": "1ο, ΙΣΤΟΡΙΚΟ – ΚΛΙΝΙΚΗ ΕΞΕΤΑΣΗ", // Το κεφάλαιο του παιχνιδιού στα ελληνικά
  "title": "Κλίμακα APGAR (APGAR Score)", // Τίτλος παιχνιδιού
  "pathophysiology": "Εδώ γράφεις την ανάλυση της παθοφυσιολογίας στα ελληνικά...",
  "fullExplanation": "Εδώ γράφεις την κλινική ερμηνεία και ανάλυση στα ελληνικά...",
  "headers": ["Κλινικό Σημείο", "0 βαθμοί", "1 βαθμός", "2 βαθμοί"], // Τίτλοι στηλών
  "rows": [
    [
      { "type": "text", "content": "Καρδιακός ρυθμός" },
      { "type": "zone", "id": "apgar_hr_0", "expected": "Άπαιρος (0)" },
      { "type": "zone", "id": "apgar_hr_1", "expected": "< 100/λεπτό" },
      { "type": "zone", "id": "apgar_hr_2", "expected": "≥ 100/λεπτό" }
    ],
    [
      { "type": "text", "content": "Αναπνοή" },
      { "type": "zone", "id": "apgar_resp_0", "expected": "Άπνοια" },
      { "type": "zone", "id": "apgar_resp_1", "expected": "Αργή, άτακτη" },
      { "type": "zone", "id": "apgar_resp_2", "expected": "Ζωηρό κλάμα" }
    ]
  ],
  "distractors": [
    "Παγίδα_1",
    "Παγίδα_2",
    "Παγίδα_3",
    "Παγίδα_4",
    "Παγίδα_5",
    "Παγίδα_6",
    "Παγίδα_7",
    "Παγίδα_8"
  ]
}
```

## 📚 PROMPT 2: Για το NotebookLM (Text-Only & Synthesis)
*Κατάλληλο για όταν του παρέχετε έγγραφα πηγών (PDF, Docs, Σημειώσεις) και θέλετε να συνθέσει ένα παιχνίδι αποκλειστικά από τις πηγές αυτές.*

### Πώς να το χρησιμοποιήσετε:
1. Ανεβάστε τα έγγραφα/πηγές σας στο NotebookLM.
2. Επικολλήστε το παρακάτω prompt στο chat του NotebookLM.

```text
Λειτουργώντας ως Εκπαιδευτής Παιδιατρικής, μελέτησε τις πηγές (sources) που σου έχω ανεβάσει.
Θέλω να επιλέξεις ή να συνθέσεις έναν σημαντικό συγκριτικό ή ταξινομικό ιατρικό πίνακα από τις πηγές αυτές και να τον μετατρέψεις σε ένα δομημένο αντικείμενο JavaScript (JSON) για ένα παιχνίδι αντιστοίχισης.

Ακολούθησε τις παρακάτω οδηγίες:

1. ΕΞΑΓΩΓΗ ΑΠΟ ΠΗΓΕΣ:
   - Βρες έναν πίνακα ή μια σύγκριση μέσα στα κείμενα (π.χ. διαφορική διάγνωση, κλινικά στάδια, βαθμολογίες κλίμακας).
   - Αν δεν υπάρχει έτοιμος πίνακας, συνέθεσε εσύ έναν συγκρίνοντας δύο ή περισσότερες οντότητες (π.χ. Ιογενής vs Βακτηριακή Μηνιγγίτιδα) βασιζόμενος ΑΠΟΚΛΕΙΣΤΙΚΑ στα δεδομένα των πηγών.

2. ΔΗΜΙΟΥΡΓΙΑ ΠΑΙΧΝΙΔΙΟΥ:
   - Απλοποίησε τη δομή ώστε να είναι κατάλληλη για πίνακα (στήλες και γραμμές).
   - Ορίστε την αριστερή στήλη ως σταθερό κείμενο ("type": "text").
   - Όλα τα άλλα κελιά πρέπει να είναι ζώνες τοποθέτησης ("type": "zone") με "id" και "expected" (τη σωστή απάντηση).

3. ΔΗΜΙΟΥΡΓΙΑ ΠΑΓΙΔΩΝ (DISTRACTORS):
   - Εξήγαγε ή δημιούργησε 8 παγίδες (distractors) που να αφορούν την παιδιατρική και το συγκεκριμένο σύστημα. 
   - Πρέπει να είναι όροι που αναφέρονται στις πηγές ή είναι κλινικά συναφείς, αλλά ΔΕΝ ταιριάζουν σε κανένα κενό του συγκεκριμένου πίνακα. Πρέπει να λειτουργούν ως πειστικά «αντίπαραδείγματα».

4. ΠΑΘΟΦΥΣΙΟΛΟΓΙΑ & ΑΝΑΛΥΣΗ:
   - Γράψε μια παράγραφο "pathophysiology" (Ελληνικά) βασισμένη στις εξηγήσεις των πηγών για τον μηχανισμό της νόσου.
   - Γράψε ένα κείμενο "fullExplanation" (Ελληνικά) που να ανακεφαλαιώνει τα κλινικά συμπεράσματα και τις οδηγίες από τις πηγές.

5. ΜΟΡΦΟΠΟΙΗΣΗ (JSON):
   - Δώσε μου το αποτέλεσμα αυστηρά στη μορφή του παρακάτω προτύπου JSON:

{
  "id": "μοναδικό_id_στα_αγγλικά",
  "chapter": "κεφάλαιο_πηγής",
  "title": "Τίτλος Πίνακα από την Πηγή",
  "pathophysiology": "Παθοφυσιολογική ανάλυση βασισμένη στις πηγές...",
  "fullExplanation": "Κλινική ερμηνεία βασισμένη στις πηγές...",
  "headers": ["Στήλη_1", "Στήλη_2", "Στήλη_3"],
  "rows": [
    [
      { "type": "text", "content": "Σταθερό κείμενο" },
      { "type": "zone", "id": "id_1", "expected": "Σωστή απάντηση από πηγή" }
    ]
  ],
  "distractors": [
    "Παγίδα_1", "Παγίδα_2", "Παγίδα_3", "Παγίδα_4", 
    "Παγίδα_5", "Παγίδα_6", "Παγίδα_7", "Παγίδα_8"
  ]
}
```
